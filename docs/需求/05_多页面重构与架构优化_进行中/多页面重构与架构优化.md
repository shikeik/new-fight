# 多页面重构与架构优化

## 背景

当前项目 `new-fight` 是单页面应用（`index.html` 直接挂载格斗游戏），但随着后续要接入坦克大战等新游戏模块，需要改造成多入口、可扩展的页面结构。同时清理现有架构中"牵一发动全身"的硬编码和相对路径地狱。

---

## 已完成

### 1. WebGL 后端切换（性能修复）
- **问题**：桌面端 WebGPURenderer 帧率极低（30~40fps），手机端满帧
- **根因**：Three.js WebGPURenderer 的 UBO 系统在大量 draw call 场景下有已知性能缺陷（GitHub #30560）
- **处理**：切回 `WebGLRenderer`，关闭 `antialias`，新增 `showRain: false` 配置
- **状态**：✅ 已完成并合并到 `main`

### 2. eruda-indexeddb CDN 支持
- **问题**：`@shikeik/eruda-indexeddb` 只有 ESM 构建，无法通过 jsdelivr CDN 直接 `<script>` 引入
- **处理**：在 `libs/eruda-indexeddb` 仓库新增 Rollup UMD/IIFE 构建，发布 `v1.0.2`，全局变量 `erudaIndexedDB`
- **状态**：✅ 已完成并 publish

### 3. 基础多页面骨架（`multi-page` 分支）
- 格斗游戏源码从 `src/*` 搬迁到 `src/fighter/*`
- 新建 `pages/fighter.html` 和 `pages/tank.html`
- 新建 `src/tank/main.ts` 空入口
- 新建导航首页 `index.html`
- Vite 配置改为三入口：`index` / `fighter` / `tank`
- **状态**：✅ 已完成（在 `multi-page` 分支，待继续重构后合并）

## 待办任务

### 任务 1：提取 `src/shared/` 公共基础设施

**目标**：把通用模块从 `src/fighter/` 中解耦出来，供 `fighter`、`tank` 及后续页面共享。

**迁移清单**：

| 原路径 | 新路径 |
|--------|--------|
| `src/fighter/lib/eval-engine.ts` | `src/shared/lib/eval-engine.ts` |
| `src/fighter/infra/mod-tick.ts` | `src/shared/infra/mod-tick.ts` |
| `src/fighter/infra/log-capture.ts` | `src/shared/infra/log-capture.ts` |
| `src/fighter/types/three.d.ts` | `src/shared/types/three.d.ts` |

**注意**：`src/fighter/types/game.ts` 是格斗游戏专属类型（`dodgeSpeed`、`justWindow`、`showRain` 等），**不移到 shared**。

**关联修改**：
- `configs/api-eval-plugin.ts` 中的 `eval-engine.ts` 引用改为 `../src/shared/lib/eval-engine.ts`（或 alias）
- `src/fighter/main.ts` 中引用 shared 模块的路径更新

---

### 任务 2：全局启用 `@/` 别名，消灭相对地狱

**目标**：所有 import 统一使用 `@/` 绝对路径，避免文件移动时频繁修改相对路径。

**当前问题**：`src/fighter/` 内部全是 `./config/...`、`../game/...` 等相对路径，没有一个人在使用已经配好的 `@/*` alias。

**改造范围**：
- `src/fighter/**/*.ts` 全部替换为 `@/fighter/xxx` 风格
- `src/fighter/main.ts` 引用 shared 时改为 `@/shared/xxx`
- `configs/api-eval-plugin.ts` 引用 eval-engine 时改为 `@/shared/lib/eval-engine.ts`

**示例**：
```ts
// 改造前
import { CFG } from "../config/game-config.ts"
import { audio } from "../audio/audio-engine.ts"

// 改造后
import { CFG } from "@/fighter/config/game-config.ts"
import { audio } from "@/fighter/audio/audio-engine.ts"
```

**约束**：不新增额外的 `@fighter/*` 等 alias（避免额外维护成本），全部用现有 `@/*`。

---

### 任务 3：Vite 入口自动扫描 `pages/*.html`

**目标**：新增/删除页面时，**不再手动修改 `configs/vite.config.ts`**。

**当前问题**：`rollupOptions.input` 是手动对象：
```ts
input: {
  index: resolve(rootDir, "index.html"),
  fighter: resolve(rootDir, "pages/fighter.html"),
  tank: resolve(rootDir, "pages/tank.html"),
}
```

**方案**：使用 Node.js 内置 API 扫描 `pages/` 目录下的 `.html` 文件，动态生成 input。

**预期效果**：
- 新建 `pages/xxx.html` + `src/xxx/main.ts` 后，Vite 自动识别
- 删除页面时同理，无需改配置

---

### 任务 4：首页导航卡片自动渲染（基于 `src/xxx/page.json` 约定）

**目标**：新增/删除页面时，**不再手动修改 `index.html` 的卡片 HTML**。

**当前问题**：首页卡片是写死的静态 HTML，每加一个新页面就要手搓一张卡片。

**约定**：每个页面在 `src/xxx/page.json` 中声明元数据：

```json
{
  "icon": "🥊",
  "title": "霓虹贫民窟",
  "desc": "3D 格斗对战 · 连招系统 · 赛博黑街风格 · 实时战斗",
  "badge": "playable"
}
```

**首页渲染**：`src/nav/main.ts` 作为导航页入口，读取所有 `src/*/page.json`，动态生成 `.nav-grid`。

**预期效果**：
1. 新建 `pages/xxx.html`
2. 新建 `src/xxx/main.ts`
3. 新建 `src/xxx/page.json`
4. **完事**，Vite 入口和首页卡片全自动感应

**注意**：`index.html` 将变为轻量壳子，只加载 `src/nav/main.ts` 和基础样式。
